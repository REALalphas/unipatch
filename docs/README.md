# Unipatch Engine Documentation

Welcome to the Unipatch Engine API Reference. This engine operates as a declarative firmware deployment system utilizing a Domain Specific Language (DSL) backed by an Abstract Syntax Tree (AST). You instruct the engine sequentially what steps you require, and it constructs the output pipeline robustly using isolated temporary build steps.

## Global Functions

These global entry points serve as the main method for defining steps within your deployment script.

* [Global.pkg](Global.pkg.md) - Instantiates the core deployment engine pipeline.
* [Global.get](Global.get.md) - Retrieves resources and artifacts via direct URL or utilizing GitHub and GitLab providers.
* [Global.create](Global.create.md) - Writes newly defined formatted files (JSON, YAML, INI, text) inside the output build path.
* [Global.edit](Global.edit.md) - Recursively overrides existing object values inside formatted files (JSON, YAML, INI) without breaking its original structure.
* [Global.remove](Global.remove.md) - Instructs the pipeline to delete unneeded target directories or specific files. Functions `remove` and `del` are equivalent.
* [Global.copy](Global.copy.md) - Duplicates local files or directories within the pipeline layout utilizing glob constraints.
* [Global.move](Global.move.md) - Shifts resources internally to different workspace directories while applying optional filtering.
* [Global.rename](Global.rename.md) - Renames a specific file or directory within the output build path.

## Node Object Reference

These pages document the underlying AST Node classes generated by the global commands, detailing the chainable properties you utilize for applying specific configurations and string-matched filtering.

* [PackageContext](PackageContext.md) - Represents the `pkg()` pipeline controller allowing chained `.put()` tasks, executed via `.execute()`.
* [GetNode](GetNode.md) - Returned by `get()`. Controls fetching archives, unpacking, and extracting specific subsets of remote assets via `.only()` and `.ignore()`.
* [CreateNode](CreateNode.md) - Returned by `create()`. Governs standard generation of serialized objects into physical files.
* [EditNode](EditNode.md) - Returned by `edit()`. Specifies properties utilizing dot notation modifiers mapped recursively via `.set()`.
* [RemoveNode](RemoveNode.md) - Returned by `remove()` or `del()`. Controls destructive actions directly inside the pipeline.
* [CopyNode](CopyNode.md) - Returned by `copy()`. Represents replication behavior augmented by `.only()` and `.ignore()`.
* [MoveNode](MoveNode.md) - Returned by `move()`. Signifies layout translation behaviors constrained by `.only()` and `.ignore()`.
* [RenameNode](RenameNode.md) - Returned by `rename()`. Specifies a direct file or directory renaming operation.

## End-to-End Beginner Example

This complete example details an entire firmware deployment workflow using chainable execution paths to sequentially download packages, modify settings seamlessly, structure new items natively, and scrub excess clutter safely via a declarative pipeline format.

```typescript
import { pkg, get, create, edit, remove } from 'unipatch-engine';

async function buildFirmware() {
    console.log("Starting firmware build pipeline...");

    const deployer = pkg().put(
        // Step 1: Establish the base system foundation
        // Targeting the atmosphere repository, ensuring we grab version 1.x containing the .zip format.
        get('github:atmosphere-nx/atmosphere', {
            version: '1.x',
            assetPattern: 'atmosphere-*.zip'
        })
        .unpack()
        .ignore('*.md'), // Intentionally filter out any markdown files.

        // Step 2: Grab specific supplementary content directly via a URL
        get('https://example.com/mods/cool-mod-v2.zip')
            .unpack()
            .only('**/*.nro'), // Ensure strictly .nro files copy successfully.

        // Step 3: Insert custom startup variables via INI serialization natively.
        create('boot/splash.ini', {
            Splash: {
                enabled: true,
                timeout: 5000
            }
        }, { type: 'ini' }),

        // Step 4: Inject modifications into a pre-existing YAML architecture
        // This configuration file extracted natively from the initial Step 1 repository payload.
        edit('config/system_settings.yaml')
            .set('performance.overclock', true)
            .set('ui.theme', 'dark'),

        // Step 5: Clean up generic template directories generated via prior steps.
        remove('config/templates')
    );

    // After all step definitions are mapped correctly, process the pipeline sequentially.
    await deployer.execute();

    console.log("Build pipeline finished successfully! Ensure you verify the 'out/' target directory.");
}

buildFirmware().catch(console.error);
```
